春晚刘谦魔术的模拟程序

昨晚春晚上刘谦的两个魔术表演都非常精彩，尤其是第二个魔术，他演绎了经典的约瑟夫环问题！ 什么是约瑟夫环问题？ 约瑟夫环（Josephus problem）是一个经典的数学问题，最早由古罗马历史学家弗拉维奥·约瑟夫斯提出，但它的名字是在19世纪由德国数学家约瑟夫·乔瑟夫斯（Josef Stein）命名的。 问题的描述是这样的：假设有n个人（编号从1到n）站成一个圆圈，从第一个人开始报数，报到某个数字（例如k）的人就被杀死，然后从下一个人开始重新报数并继续这个过程，直到只剩下一个人留下来。 问题的关键是找出存活下来的那个人的编号。 结合扑克牌解释约瑟夫环问题 1、考虑最简单的情况 假设有2张牌，编号分别是1和2。 首先将1放到后面，扔掉2。剩下的就是最开始放在最上边的那张1。 2、稍微复杂一点的情况，牌的张数是2的n次方 比如有8张牌，编号分别是1、2、3、4、5、6、7、8。 第一轮会把2、4、6、8扔掉，剩下1、3、5、7按顺序放在后面，又退化成了4张牌的情况。 第二轮会把3、7扔掉，剩下1、5按顺序放在后面，又退化成了2张牌的情况。 第三轮把5扔掉，剩下1，就是最初在最前面的那张。 结论：如果牌的张数是2^n，最后剩下的一定是最开始放在牌堆顶的那张。 3、考虑任意的情况，牌的张数是2^n+m 比如牌的张数是11，等于8+3。把1放到后面，把2扔掉，把3放到后面，把4扔掉，把5放到后面，把6扔掉，现在剩下的编号序列是7、8、9、10、11、1、3、5，这又是8张牌的情况！最后一定剩下的是现在牌堆顶的7！ 因此，只要提前知道牌的张数，就一定能马上推导出最终是剩下哪一张牌。一切的魔法都是数学！！都是算法！！ 见证奇迹的时刻！魔术的流程 4张牌对折后撕开，就是8张，叠放在一起就是ABCDABCD。注意，ABCD四个数字是完全等价的。 根据名字字数，把顶上的牌放到下面，但怎么放都不会改变循环序列的相对位置。譬如2次，最后变成CDABCDAB；譬如3次，最后换成DABCDABC。但无论怎么操作，第4张和第8张牌都是一样的。 把顶上3张插到中间任意位置。这一步非常重要！因为操作完之后必然出现第1张和第8张牌是一样的！以名字两个字为例，可以写成BxxxxxxB（这里的x是其他和B不同的牌）。 拿掉顶上的牌放到一边，记为B。剩下的序列是xxxxxxB，一共7张牌。 南方人/北方人/不确定，分别拿顶上的1/2/3张牌插到中间，但是不会改变剩下7张牌是xxxxxxB的结果。 男生拿掉1张，女生拿掉2张。也就是男生剩下6张，女生剩下5张。分别是xxxxxB和xxxxB。 循环7次，把最顶上的放到最底下，男生和女生分别会是xxxxBx和xxBxx。 最后执行约瑟夫环过程！操作到最后只剩下1张。当牌数为6时（男生），剩下的就是第5张牌；当牌数为5时（女生），剩下的就是第3张牌。Bingo！就是第4步拿掉的那张牌！ 下面是完整的 JavaScript 代码实现： javascript 代码解读 复制代码 // 定义一个函数，用于把牌堆顶n张牌移动到末尾 function moveCardBack(n, arr) { // 循环n次，把队列第一张牌放到队列末尾 for (let i = 0; 小结一下年底找工作现状

本人前端开发，2年9个月工作经验，技术栈vue全家桶+微信小程序原生开发，会一点thinkphp和nodejs。 上家公司因为公司不盈利，老板解散了我们的4人技术小组，12月15日离职，离职前一周开始找工作，截止发稿已经10天，这里小结一下找工作现状。 首先本人已经有4家公司的经历了，算是简历上的一大败笔吧，至于频繁换工作的原因，基本上都是公司倒闭或者公司业绩不好（真的真的真的）。然后找工作的方式基本上都是在boss直聘上直投，屏蔽了所有外包公司，只投附近的，有一点规模的公司。可能是因为年底，附近的前端岗位，没有之前金九银十那会儿看到的多，但是也还是陆陆续续能看到一些公司发布的招聘信息。 这些天投了将近50几家，收到发简历邀请的大概有20家左右，总共收到的面试邀请只有5家，目前已经面试完最后一家。 首先第一家公司面试，技术面试不太理想，准备的不充分，面试官问的很多简单的问题，没有答对，或者答得磕磕巴巴，但是面试官可能觉得还凑合，所以叫来了技术主管，技术主管只问了我开发中遇到的最难的问题是什么？可能我答得文不对题吧，这后来也就没有了回复，在boss上询问也是没回复； 第二家面试有点远，而且是单休，面试官比较亲和，是个和我年纪差不多的前端吧（ps：但是好像技术比较好的样子），这家的面试稍微有点准备了，面试的问题也答出来了个七七八八，周五去面的，周一晚上6点收到人事打来的电话offer，最后因为觉得单休，工资又没有达到预期， 所以拒绝了。 第三家和第四家里的很近，第三家是周日约的电话面试，然后电话面试个人觉得面的很糟糕，中间几个问题卡住了好久还是没答上来，但是没想到周一时候，人事在boss上联系我说让我去公司复试，复试过程就是先拿来一张笔试题给我做，做完以后，和技术主管进行了半小时左右的电话面试，可能是技术面的结果不好，电话面没有问技术相关的问题，问了一下个人规划的问题，可能答得面试官不是很满意，也就没有后续了(后来还询问了一下，结果是未通过)。 第四家是当天下午去的，先是和人事简单的做了一下自我介绍，然后技术主管面了15分钟左右，结果是当场被人事告知，技术面没有通过（心酸）。 第五家是个比较大的上市公司，开了一个子公司，新项目需要招前端开发，个人感觉这家公司很有逼格，没进过大公司，有点激动。周二下午五点进行了电话面试后，周三约了现场面试，技术面了50分钟左右，人事没面，直接叫我回去等公司（八成又是凉），但是个人感觉这家是表现的最好的了，面试过程中也没有很紧张。猜测：八成是简历不太好看，大公司可能不太接受。 总结：面试的五家公司，除了第一家准备不充分以外，其他几家面试成绩个人给个60分到80分之间吧，至于为什么都凉了，可能是面试的过程中，有一些简单的面试题，答得不符合面试官的期望（其实就是菜QAQ），另一方面，可能就是简历上的4家工作经历太扎眼了吧。 最后，面试的问题基本上是框架为主（vue）约60%，原生js占了40%吧，css和html很少。 定一个小目标，过年之前一定要找到工作!!!
